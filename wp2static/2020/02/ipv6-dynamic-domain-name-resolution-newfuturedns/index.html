ipv6动态解析域名NewFutureDDNS

2020年2月28日 281点热度

注意:把TTL时间改短,否则就算提交ddns服务商更新了,但本地dns服务器缓存还没过期,导致本地访问时dns解析错误

IPV6支持的APP

  • windows支持的(mstsc)
  • andorid支持的(nplayer,mstsc)
  • android不支持(es文件浏览器;juicessh).

体验:

  • 2020年2月25日21:32:29
    小米平板4>移动4g热点(mix2)>联通广西钉钉(红米note5)热点>新3路由器padavan>z7m笔记本电脑:

    • 远程桌面可以实际使用,wiz可以编辑,不算很卡,可接受的延时.(客户端设备显示网络速度在300KB上限大约.资源管理器cpu页面全屏时100KB流速.)

域名同时拥有ipv4和ipv6解析

  • z7m同时提交本机公网v6和本机内网v4的ip到动态域名,外网访问动态域名情况下,优先v4,因为v4无法访问,然后解析v6访问.(安卓和win的mstsc测试通过v4+v6的动态域名成功连接v6公网).

客户端NewFutureDDNS + CloudFlare API

https://github.com/NewFuture/DDNS/blob/master/README.md

  • CloudFlare
    ** id为cf登录的邮箱地址,token使用Global API Key.(My Profile>API Tokens>Global API Key).
    ** cf不支持子域名.只能顶级域名example.com,不支持subdomain.example.com.
    ** 修改DNS生效时间:5分钟.(可能长2小时至24小时)
    ** 不需要在cf中预配动态域名,客户端通过API提交配置后会自动根据配置添加域名.
  • NewFutureDDNS
    • 计划任务的规则
      只有在和上次提交的内容有变更的情况下,客户端才会向DDNS服务商提交最后申请.因此测试计划任务的时候,需要变化ip,或者变化config文件.
    • 重启后未登录也能自动执行的计划任务配置:
      • 常规-使用平常登录的用户名权限
      • 常规-[不管用户是否登录都要运行]
      • 常规-[使用最高权限运行]
      • 设置-如果此任务已经运行,以下规则使用:[停止现有实例].否则过去任务因为某种原因一直在运行没被kiil的话,下一个时间就无法正常执行功能.

使用问题

  • 安卓端访问动态ipv6,不管是dynv6还是cloudflare 的ipv6都需要失败一次访问 第2次才能成功访问,手机和平板皆是如此. 猜测原因为第一次默认的v4解析失败才会尝试v6解析. //2020-12-21
